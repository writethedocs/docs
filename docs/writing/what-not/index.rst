##################################
What NOT to write in documentation
##################################

Here are tips about **content you should NOT put in documentation**, i.e.
content you'd better put somewhere else.

Documentation is a communication channel for development team or users, just
like email or bugtracker. It is suitable to receive, store and send some
information. But it is not meant to receive it all.

**Redirect information to the right communication channel.**

The following patterns have poor value in documentation:

.. toctree::

   scripts
   code-api
   configuration
   templates
   todo
